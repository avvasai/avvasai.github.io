Coursera for Robots — Capturing multi-modal skill datasets for robotic foundation models

Building a dynamic catalogue of skills that robots can pull and do test-time training on during deployment. We capture student-teacher interaction datasets to teach robots how to learn skills and interact with humans — unlike existing data companies that focus on in-the-wild egocentric footage.

As a co-founder, I designed and built the hardware and video processing pipeline for multi-modal data capture. Created a reproducible recording setup using DSLRs, Intel RealSense, webcams, ESP32, head mounts, and uniform lighting systems. Also contributed to AI pipeline setup for faster processing.

Problem Statement

  • Need: Robotic foundation models require high-quality, multi-modal training data to learn dexterous manipulation skills — but physical AI lacks internet-scale datasets
  • Prior art: Existing robotics data companies (Micro1, Scale, Surge, MicroAGI, BuildAI) focus primarily on in-the-wild egocentric footage, missing critical modalities like gripping force, temperature sensing, and structured student-teacher interactions
  • Open question: Could custom economical hardware matching target robot sensory inputs, combined with structured skill demonstrations, provide significantly higher-value post-training data for physical AI?

Designed a reproducible multi-camera capture system integrating professional motion capture with consumer-grade sensors. The setup is engineered to be portable and deployable across our partner gait labs in India.

Stack Complete tech stack
  • Motion Capture: Optitrack for precise full-body pose tracking
  • Egocentric: Head-mounted dashcams for first-person perspective
  • Depth: Intel RealSense for RGB-D capture
  • Video: DSLRs and webcams for multi-angle coverage
  • Sensors: ESP32 for gripping force and temperature data
  • Processing: Claude Code for pipeline automation, HaWoR for 3D hand pose reconstruction
Sensors ESP32 sensor integration
  • Gripping Force: Force-sensitive resistors integrated into tool handles for measuring grip pressure during manipulation tasks
  • Temperature: Thermocouples for capturing thermal data relevant to soldering and heat-sensitive tasks
  • Synchronization: ESP32 timestamps all sensor readings for alignment with video streams
  • Wireless: Data transmitted via WiFi to central recording system

Data Capture System

Capturing student-teacher interaction datasets to teach robots how to learn skills and interact with humans. Our unique approach focuses on structured instruction rather than in-the-wild egocentric footage.

Key 7 unique data modalities
  • Gripping Force: Measuring grip pressure during manipulation tasks
  • Temperature Sensing: Thermal data for heat-sensitive tasks like soldering
  • Chain-of-thought Audio: Self-annotation through verbal explanation during tasks
  • Full Body Pose: Complete skeletal tracking via Optitrack
  • Student-Teacher Interactions: Structured learning demonstrations
  • Exception Cases: Captured task failures for robust training
  • Multi-modal Instructions: PDFs, assembly videos paired with demonstrations

Egocentric Capture: Head-mounted dashcams capture first-person perspective essential for training robots on human visual perception during task execution. Uniform lighting systems ensure consistent image quality across all recording sessions.

Processing Pipeline

Established the video recording and processing pipeline for handling multi-view synchronized footage. Integrated HaWoR for 3D hand pose reconstruction and contributed to AI pipeline optimization for faster processing throughput.

Pipeline Processing workflow
  • Synchronization: Multi-view video streams aligned using audio sync and timestamp matching
  • Pose Estimation: HaWoR integration for accurate 3D hand pose reconstruction
  • Annotation: Chain-of-thought audio enables self-annotation of skill demonstrations
  • Automation: Claude Code for pipeline automation and quality checks
